# 使用场景

这页按真实使用方式整理 LuckyHarness 适合承担的角色。

## 1. 本地 Agent 开发调试

适合人群：

- Agent 开发者
- Prompt / tool 调试者
- 需要快速验证模型行为的人

推荐入口：

```bash
lh chat
lh chat "帮我总结这个仓库"
```

能做什么：

- 调整 SOUL 和 system prompt
- 验证 provider/model 配置
- 调试工具调用
- 观察 RAG 和 memory 是否按预期注入
- 快速复现用户输入和模型响应

## 2. 项目知识库问答

适合场景：

- 给代码库文档做问答入口
- 给部署手册做检索
- 给团队规范做知识库
- 给个人 Obsidian/Markdown 笔记做问答

流程：

```bash
lh rag index ./docs
lh rag search "怎么部署 NapCat 网关"
lh chat "根据知识库说明，帮我排查部署失败"
```

价值：

- 降低查文档成本
- 把项目背景注入 Agent
- 让 Agent 回答更贴近当前仓库和团队上下文

## 3. Telegram 机器人

适合场景：

- 个人 AI 助手
- 群组问答机器人
- 日常任务入口
- 远程触发 Agent 工作流

启动：

```bash
lh msg-gateway start --platform telegram
```

可以承载：

- 普通聊天
- RAG 问答
- cron 结果通知
- 图片/媒体处理
- 工具调用进度展示
- 会话继续追问

## 4. QQ 官方机器人

适合场景：

- QQ 频道/群机器人
- 中文社群助手
- 群内问答和自动化入口

启动：

```bash
lh msg-gateway start --platform qqofficial
```

常见用法：

- 群内问答
- 指令式操作
- 进度追踪
- 与 LuckyHarness session 绑定
- 接入记忆和 RAG

## 5. NapCat / OneBot QQ 接入

适合场景：

- 使用 NapCat 登录 QQ 账号
- 通过 OneBot v11 reverse WebSocket 接入 LuckyHarness
- 不走 QQ 官方机器人能力，而是复用本地 QQ 客户端环境

LuckyHarness 监听：

```bash
lh msg-gateway start --platform napcat --napcat-listen 0.0.0.0:6701
```

NapCat 连接：

```text
ws://<luckyharness-host>:6701/onebot/v11/ws
```

适合长期在线 QQ 助手、群内问答、私聊助手和本地网络部署。

## 6. 个人微信助手

适合场景：

- 使用 iLink Bot API 接入个人微信
- 构建私聊助手
- 将微信消息转入 LuckyHarness runtime

启动：

```bash
lh msg-gateway start --platform weixin
```

登录辅助：

```bash
lh msg-gateway weixin-login
```

建议先用 allowlist 控制入口，避免开放到不期望的会话。

## 7. 团队内部 Agent API

适合场景：

- 内部系统调用 Agent
- Web 应用后端接入
- 自动化脚本调用
- 给 Dashboard/TUI 提供统一服务

启动：

```bash
lh serve --addr 0.0.0.0:9090
```

健康检查：

```bash
curl http://127.0.0.1:9090/api/v1/health
```

注意事项：

- 生产环境要限制 CORS 和网络访问
- 配置 rate limit
- API key 和配置文件不要提交进仓库
- 容器部署要确认 `HOME` 和配置挂载路径

## 8. 定时任务与自动化助手

适合场景：

- 每天总结信息
- 定期检查网页或文档
- 定时触发提醒
- 后台执行长任务
- 将结果发回 Telegram/QQ

LuckyHarness 的 cron/autonomy 能让 Agent 从一次性聊天变成持续运行的助手。

典型任务：

- 每天早上整理项目状态
- 定期检查服务健康
- 定时抓取信息并总结
- 自动执行长任务并通知结果

## 9. Agent 能力评测与实验

适合场景：

- 评估工具路由策略
- 评估记忆激活效果
- 评估 prompt cache 命中
- 评估多 Agent 编排
- 做实验报告和回归测试

仓库包含多类 benchmark 命令：

```text
cmd/lh-cache-bench
cmd/lh-context-packer-bench
cmd/lh-memory-bench
cmd/lh-multiagent-bench
cmd/lh-tool-bench
```

这类场景更适合研究和工程优化，不一定是普通用户第一天就需要使用的入口。

## 10. 推荐选择

| 目标 | 推荐入口 |
|---|---|
| 快速试用 | `lh chat` |
| 做 API 服务 | `lh serve` |
| 做个人机器人 | Telegram / Weixin |
| 做 QQ 机器人 | QQ Official / NapCat |
| 做知识库问答 | RAG + `lh chat` / API / 网关 |
| 做长期任务 | Cron + Autonomy |
| 做工程实验 | benchmark 命令 |
