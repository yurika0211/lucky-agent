# 特色功能

LuckyHarness 的定位是长期运行的 Agent runtime，而不是单轮聊天 demo。它把模型、工具、记忆、知识库、消息入口、API 和部署流程放到一套可运维的结构里。

## 1. 统一运行时

LuckyHarness 的多个入口共享同一个 Agent 核心：

| 入口 | 用途 |
|---|---|
| `lh chat` | 本地 REPL 和单轮调试 |
| `lh serve` | HTTP API、Dashboard、WebSocket 和集成入口 |
| `lh msg-gateway start` | Telegram、QQ、NapCat、Weixin 等消息入口 |
| TUI | 终端工作区和长会话交互 |
| GUI Dashboard | 运行状态、会话和活动流观察 |

统一运行时的好处是：本地验证、API 接入、消息机器人和容器部署不需要维护多套逻辑。

## 2. 配置优先的运行模型

核心行为集中在：

```text
${HOME}/.luckyharness/config.json
```

这让源码运行、Docker 开发环境、生产容器和消息网关共享同一套配置模型。

常见配置维度包括：

- provider、api_key、api_base、model
- server.addr、CORS、rate limit、日志格式
- Telegram / QQ / NapCat / Weixin 网关
- RAG、memory、embedding
- Cron、Autonomy、工具和 OpenCLI

## 3. 长期记忆系统

LuckyHarness 不只处理眼前一轮对话，也会围绕长期使用维护记忆。

默认运行目录中包含：

```text
memory/
├── 00_Index/
├── 10_Profile/
├── 20_Projects/
├── 30_Sessions/
├── 40_Decisions/
├── 50_Facts/
├── 60_Rules/
├── 70_Trajectories/
└── 90_Archive/
```

适合保存：

- 用户偏好
- 项目背景
- 决策记录
- 可复用事实
- 会话轨迹
- 长期任务状态

## 4. RAG 知识库

LuckyHarness 提供 RAG 检索和知识注入能力。

常用命令：

```bash
lh rag index ./docs
lh rag search "deployment"
```

可用于：

- 项目文档问答
- 运维手册检索
- 代码库说明检索
- 私有知识库检索
- 长期上下文增强

## 5. 多平台消息网关

LuckyHarness 当前重点支持：

- Telegram
- QQ Official
- NapCat / OneBot v11 reverse WebSocket
- Weixin iLink Bot API
- OpenClaw Weixin 兼容入口

同一个 Agent 可以接入多个外部入口。消息网关侧会处理平台消息、会话、回复格式、媒体和命令入口，再交给统一 Agent runtime。

## 6. TUI 和 GUI Dashboard

TUI 适合开发者在终端内长期使用：

- 持续聊天
- 切换会话
- 调用常用 `lh` 命令
- 观察流式输出

GUI Dashboard 适合运行态观察：

- 会话状态
- 活动流
- 网关状态
- Dashboard API 联动

## 7. 工具系统与 OpenCLI

LuckyHarness 的工具调用围绕统一 tool service 组织，内置能力包括：

- 文件系统相关工具
- Web 内容抽取
- 搜索和查询
- RAG index/search
- memory remember/recall
- cron/autonomy/heartbeat
- terminal/shell 兼容工具
- 多模态处理能力

OpenCLI 可作为外部网页和平台操作入口，适合把真实网站能力接入 Agent。

## 8. Cron、Autonomy 和后台任务

LuckyHarness 支持把 Agent 从“被动回答”扩展到“长期在线执行”。

典型能力：

- 定时任务
- 后台 worker
- 队列持久化
- 结果通知
- cron 结果接回会话
- autonomy 任务生命周期管理

适合提醒、巡检、定期总结、自动采集和持续处理任务。

## 9. 多 Agent 与 Benchmark

仓库中包含多类 benchmark 命令，用于评估和改进 Agent 能力：

- prompt cache benchmark
- memory activation benchmark
- context packer benchmark
- tool tuning benchmark
- multi-agent benchmark

这部分适合做 Agent 行为评估、工具路由实验、记忆激活实验和多 Agent 编排研究。

## 10. 部署友好

LuckyHarness 支持：

- 源码运行
- 开发 Docker Compose
- 生产 Docker Compose
- GitHub Releases 发布的预构建二进制包
- systemd 风格长期运行
- 多 profile 网关部署

部署时关键是守住同一个约定：

```text
HOME=/var/lib/luckyharness
config.json -> /var/lib/luckyharness/.luckyharness/config.json
```
