# 使用指南

这页介绍 LuckyHarness 从初始化到启动各类入口的基本流程。

## 环境要求

- Go 环境，项目当前通过 `go run ./cmd/lh ...` 直接运行。
- 可用的 OpenAI-compatible 模型服务，或其他已配置 provider。
- 如需 GUI/TUI 前端开发，需要 Node.js 和 npm。
- 如需 Docker 部署，需要 Docker 和 Docker Compose。

## 1. 初始化运行目录

LuckyHarness 默认读取：

```text
${HOME}/.luckyharness/config.json
```

先执行初始化：

```bash
go run ./cmd/lh init
```

初始化后会生成类似结构：

```text
~/.luckyharness/
├── config.json
├── SOUL.md
├── sessions/
├── memory/
├── logs/
├── skills/
├── rag/
├── workspace/
├── runtime/
└── data/
```

## 2. 配置模型

最小配置通常包括 provider、API key、API base 和模型名。

```bash
go run ./cmd/lh config set provider openai
go run ./cmd/lh config set api_key sk-your-api-key
go run ./cmd/lh config set api_base https://api.openai.com/v1
go run ./cmd/lh config set model gpt-5.4-mini
```

查看配置：

```bash
go run ./cmd/lh config list
go run ./cmd/lh config get model
```

## 3. 本地聊天调试

单轮调用：

```bash
go run ./cmd/lh chat "Summarize this repository"
```

进入 REPL：

```bash
go run ./cmd/lh chat
```

适合用来调试 prompt、工具调用、RAG、记忆和模型配置。

## 4. 启动 HTTP API

```bash
go run ./cmd/lh serve --addr 127.0.0.1:9090
```

健康检查：

```bash
curl http://127.0.0.1:9090/api/v1/health
```

如果要让容器或其他机器访问，配置里的 `server.addr` 通常需要设为：

```text
0.0.0.0:9090
```

## 5. 启动 GUI Dashboard

Dashboard 适合观察运行状态、会话和活动流。

源码方式通常先启动 API：

```bash
go run ./cmd/lh serve --addr 127.0.0.1:9090
```

前端开发模式：

```bash
cd UI
npm install
npm run dev --workspace GUI
```

构建 GUI：

```bash
cd UI
npm run build --workspace GUI
```

Go dashboard 会在存在 `UI/GUI/dist` 时优先服务该目录。

## 6. 启动 TUI

TUI 适合在终端里长期对话、切换会话和执行常用命令。

```bash
cd UI
npm run dev --workspace TUI -- --api-base http://127.0.0.1:9090 --session dashboard-main
```

或使用项目 Makefile：

```bash
make tui
```

## 7. 启动消息网关

### Telegram

配置：

```bash
go run ./cmd/lh config set msg_gateway.telegram.token your-telegram-bot-token
```

启动：

```bash
go run ./cmd/lh msg-gateway start --platform telegram
```

### QQ 官方机器人

关键字段：

- `msg_gateway.qqofficial.app_id`
- `msg_gateway.qqofficial.app_secret`
- `msg_gateway.qqofficial.sandbox`
- `msg_gateway.qqofficial.allowed_chats`
- `msg_gateway.qqofficial.allowed_users`

启动：

```bash
go run ./cmd/lh msg-gateway start --platform qqofficial
```

### NapCat / OneBot v11

LuckyHarness 作为反向 WebSocket 服务端，NapCat 主动连接进来。

```bash
go run ./cmd/lh config set msg_gateway.napcat.listen_addr 127.0.0.1:6701
go run ./cmd/lh config set msg_gateway.napcat.path /onebot/v11/ws
go run ./cmd/lh msg-gateway start --platform napcat
```

跨机器部署时，监听地址通常要改成：

```bash
go run ./cmd/lh config set msg_gateway.napcat.listen_addr 0.0.0.0:6701
```

### Weixin

LuckyHarness 支持个人微信 iLink Bot API 入口。

关键字段：

- `msg_gateway.weixin.token`
- `msg_gateway.weixin.account_id`
- `msg_gateway.weixin.base_url`
- `msg_gateway.weixin.dm_policy`
- `msg_gateway.weixin.group_policy`

启动：

```bash
go run ./cmd/lh msg-gateway start --platform weixin
```

登录辅助：

```bash
go run ./cmd/lh msg-gateway weixin-login
```

## 8. RAG 知识库

索引目录：

```bash
go run ./cmd/lh rag index ./docs
```

搜索：

```bash
go run ./cmd/lh rag search "deployment"
```

RAG 可用于给 Agent 注入项目文档、个人笔记、运维手册和长期知识。

## 9. Docker 部署

开发环境：

```bash
docker compose up -d --build luckyharness
```

生产环境：

```bash
docker compose -f docker-compose.prod.yml up -d
```

启用 Telegram profile：

```bash
docker compose -f docker-compose.prod.yml --profile telegram up -d
```

启用 NapCat profile：

```bash
docker compose -f docker-compose.prod.yml --profile napcat up -d
```

容器部署时最重要的是确认：

- `HOME` 指向持久化目录，例如 `/var/lib/luckyharness`
- `config.json` 挂载到 `${HOME}/.luckyharness/config.json`
- `server.addr` 是否允许容器内外访问

## 10. 常用排障

配置没生效：

- 检查当前进程的 `HOME`
- 检查 `${HOME}/.luckyharness/config.json`
- 容器里确认挂载路径是否正确

消息网关连不上 API：

- 检查 `server.addr`
- 容器跨服务访问不要使用错误的 `127.0.0.1`
- 检查防火墙和端口映射

NapCat 连接失败：

- 确认 LuckyHarness 已启动 NapCat 网关
- 确认 NapCat URL 路径和端口一致
- 跨机器部署不要监听 `127.0.0.1`

模型调用失败：

- 检查 `provider`
- 检查 `api_key`
- 检查 `api_base`
- 检查 `model`
- 查看 `~/.luckyharness/logs`
